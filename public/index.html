What is Vorexis-Claw?

Vorexis-Claw is an autonomous software engineer AI — a terminal-native platform for building, planning, and understanding codebases. Single-prompt architecture: describe what you want, and it automatically routes to Ask, Plan, or Agent engines.

Core Philosophy

  • No mode menus — one prompt does everything
  • Everything runs in the terminal — no browser required
  • Voice and text share the same intelligent pipeline
  • Session memory carries context across follow-ups

Requirements

Bun 1.0+, Git (recommended). One of: OpenRouter API key, Ollama, or LM Studio.

Installation

git clone https://github.com/ajeetpandit123/Vorexis-claw.git
cd Vorexis-claw && bun install && bun run build
bun link

Cloud Setup (OpenRouter)

vorexis-claw login

Local Setup (Ollama — no API key)

ollama pull qwen2.5:7b
vorexis-claw provider set ollama

Health Check

vorexis-claw doctor

Starting a Session

vorexis-claw
vorexis-claw wakeup

On startup you see project context: framework, language, git branch, and status. Then the prompt: What would you like to build, fix, understand, or plan?

Example Prompts

  • What's inside README.md?
  • Explain how the agent orchestrator works
  • Create a roadmap for a MERN ecommerce app
  • Build JWT authentication with refresh tokens
  • Review my open pull requests

Keyboard Shortcuts

  • Tab — Start voice recording (mic button in composer)
  • ENTER — Submit typed prompt / stop recording
  • ESC — Cancel voice recording
  • Ctrl+C — Exit session

Intent Routing

You never pick a mode. The system routes automatically based on your prompt:

  • Ask — Questions, explain, what's inside → read-only Q&A
  • Plan — Plan, roadmap, architecture, strategy → step-by-step execution
  • Agent — Build, fix, create, implement, PR/issue → autonomous edits + approval

Flow: User Input → Intent Detection → Engine → Platform Tools → Response → Session Memory

Composer Voice Input

Type in the composer bar or press Tab to activate the mic (same as clicking 🎤). Recording stops on ENTER. AI responses are text-only by default — enable TTS in settings if you want spoken output.

vorexis-claw settings

STT Providers

  • whisper — OpenAI Whisper (requires openaiApiKey)
  • whisper.cpp — Local, no API key
  • deepgram — Deepgram API
  • assemblyai — AssemblyAI API

TTS (opt-in)

  • edge-tts — Free, keyless
  • Set voiceOutput: false (default) to disable auto speech

Telegram Bot Setup

Create a bot via @BotFather on Telegram. Set environment variables:

TELEGRAM_BOT_TOKEN=your_bot_token
TELEGRAM_OWNER_ID=your_telegram_user_id
vorexis-claw telegram

Commands

  • /ask — Read-only codebase questions
  • /plan — Generate and run plans
  • /agent — Autonomous agent with approval

Only the configured owner ID can control the bot. Same intent routing and engines as the terminal session.

GitHub Setup

Create a token at github.com/settings/tokens with repo scope.

vorexis-claw github login
vorexis-claw github status

In-Session Prompts

  • Review my open pull requests
  • Create a PR for the authentication feature
  • Fix issue #42

MCP Integration

Edit ~/.vorexis-claw/mcp.json then:

vorexis-claw mcp list
vorexis-claw mcp connect filesystem
vorexis-claw mcp status

Model Providers

vorexis-claw provider list
vorexis-claw provider set openrouter
vorexis-claw provider status

Supported: openrouter, openai, anthropic, google, ollama, lmstudio. Auto model routing selects the best model per task type.

All CLI Commands

  • vorexis-claw / wakeup / start — Start session
  • login / logout / whoami — Authentication
  • settings — Voice configuration wizard
  • github login|logout|status — GitHub auth
  • mcp list|connect|disconnect|status — MCP servers
  • provider list|set|status — Model providers
  • doctor — Health checks
  • telegram — Start Telegram bot (requires env vars)

Slash Commands (in-session)

  • /help — Show help
  • /reset — Clear session memory
  • /exit — Exit session
